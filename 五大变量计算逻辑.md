# 商业选址五大变量纬度 · 计算逻辑详细方案

> 原型：成都市武侯区·天府国际金融中心（2km 商圈）
> 数据底座：PDF 实测数据 + AI 行业经验补齐
> 用法：每个维度下列 2-3 个公式变体（① 基础 / ② 进阶 / ③ 专业），逐条勾选你要用的那一版

---

## 总览：5 个维度 + 1 个综合输出

| 维度 | 英文 | 权重建议 | 核心问题 |
|---|---|---|---|
| ① 地理性 | Geography | 25% | 这块地"位置"值多少分？ |
| ② 经济动态性 | Economic Dynamics | 20% | 这里有多少"会花钱"的人在动？ |
| ③ 消费者画像 | Consumer Profile | 20% | 来的人会不会变成"买的人"？ |
| ④ 竞争生态层 | Competition Ecosystem | 20% | 周围是"对手"还是"队友"？ |
| ⑤ 外部性 | Externality | 15% | 不可控的风/势，是顺还是逆？ |

**综合输出（4 选 1 或全选）**：
- 综合得分（0-10）
- 预测年营业额（万元）
- 租金比（租金 / 营收）
- 投资回报率 ROI / 回收期（年）

## ① 地理性（Geography）

**含义**：物理位置先天禀赋。位置一旦定下，几乎不可逆。

### 子指标 & 数据来源

| 子指标 | 符号 | 数据来源 | 天府 IFC 取值 |
|---|---|---|---|
| 可用商业面积（㎡） | A | 业主方 / AI 估算 | 假设候选铺位 200㎡ |
| 城市地段等级（1-10） | T | 报告"区域等级" | **10**（金融城核心 CBD） |
| 最近地铁口距离（m） | d_m | PDF 数据 | 56-150m（金融城站 / 双子塔站） |
| 街道引导评分（1-10） | S | 主街宽度 + 视线可达 + 招牌权限 | 9（交子大道临街一线） |
| 楼层系数 | f_floor | 1F=1.0, 2F=0.7, B1=0.85, 3F+=0.5 | 1F=1.0 |
| 通勤动线评分（1-10） | C | 早晚高峰 OD 数据 / AI 估算 | 8.5（写字楼→地铁主轴） |

### 公式变体

**① 基础版 · 加权评分（0-10）**
```
G_score = 0.30·T + 0.25·(10 − min(d_m/50, 10)) + 0.20·S + 0.15·(f_floor×10) + 0.10·C
```
> 距离衰减：每 50m 扣 1 分，超过 500m 归零。
> 天府 IFC 算例：0.30·10 + 0.25·(10−2) + 0.20·9 + 0.15·10 + 0.10·8.5 ≈ **8.65 / 10**

**② 进阶版 · 距离衰减函数（连续）**
```
地铁可达性 R_metro = exp(−d_m / 200)      # 200m 半衰
位置评分 G = T × R_metro × S/10 × f_floor × (0.5 + 0.05·C)
最终归一化 G_score = G / G_max × 10
```
> 天府 IFC 算例：10 × exp(−0.5) × 0.9 × 1.0 × (0.5+0.425) ≈ 5.05 → 归一化 **≈ 9.1 / 10**

**③ 专业版 · GIS 缓冲区 + 街道中心性**
```
G = α·T + β·Σ(POI权重/距离²) + γ·BC + δ·f_floor
```
其中 BC = 街道介数中心性（Betweenness Centrality，从路网图算出）。
> 需要 OSM 路网数据，沙盘 demo 中可用预算好的固定值替代。

## ② 经济动态性（Economic Dynamics）

**含义**：这块地"流过来的钱量"。流量 × 钱包 × 意愿三件套。

### 子指标 & 数据来源

| 子指标 | 符号 | 数据来源 | 天府 IFC 取值 |
|---|---|---|---|
| 日均人流量（人/天） | F | 移动信令 / AI 估算 | **8.5 万**（金融城核心 500m 内） |
| 客群人均月消费（元） | P | 武侯区人均消费 32171 元/年 ÷ 12 | 2681 元/月 |
| 客群消费倾向（0-1） | π | 周边房价 30195 / 市均 1.95×推导 | 0.85（高消费力） |
| 时段集中度（0-1） | τ | 工作日午晚高峰占比 | 0.62（写字楼型，午市强） |
| 业态-客群匹配度 | M | 你的品类 × 客群画像 | 假设咖啡/简餐：0.9 |

### 公式变体

**① 基础版 · 潜在消费规模**
```
E = F × P_day × π × M
P_day = P / 30           # 月消费 → 日消费
```
> 天府 IFC 算例：85000 × (2681/30) × 0.85 × 0.9 ≈ **581 万元/天潜在盘子**
> 单店捕获率通常 0.05-0.3%，对应单日营收 0.3-1.7 万元

**② 进阶版 · 引入时段权重**
```
E = F × P_day × π × M × (1 + τ × 0.5)
```
> 高峰集中度高 → 单位时间转化效率更高，乘 1.31 修正。
> 天府 IFC 算例：581 × 1.31 ≈ **761 万元/天**

**③ 专业版 · Huff 商圈引力模型**
```
P_ij = (A_j^α / d_ij^β) / Σ_k(A_k^α / d_ik^β)
预期客流 = Σ_i (居住人口_i × P_ij × 出行频次)
预期营收 = 预期客流 × 客单价 × 转化率
```
- α=1.2（吸引力面积弹性），β=1.8（距离衰减）
- 需要把 91 个小区当作"客源点 i"，候选铺位当作"目的地 j"
> Demo 可简化为 5-10 个代表性客源点。

## ③ 消费者画像（Consumer Profile）

**含义**：来的人 ≠ 买的人。看转化漏斗。

### 子指标 & 数据来源

| 子指标 | 符号 | 数据来源 | 天府 IFC 取值 |
|---|---|---|---|
| 进店率（路过→进店） | r1 | 行业基准 / AI | 餐饮 8-15%，零售 3-8% |
| 转化率（进店→购买） | r2 | 行业基准 / AI | 餐饮 60-80%，零售 20-35% |
| 提袋率（购买→多件） | r3 | POS 数据 / AI | 1.1-1.6 件/单 |
| 客单价（元） | AOV | 业态定价 / 周边均价 | 咖啡 35，简餐 55，正餐 120 |
| 复购率（月） | r4 | CRM / AI | 写字楼客 2-4 次/月 |

### 公式变体

**① 基础版 · 单日转化漏斗**
```
日营收 = F_pass × r1 × r2 × r3 × AOV
```
> 天府 IFC 算例（咖啡店）：1500（铺位前路过）× 0.12 × 0.7 × 1.2 × 35 ≈ **5292 元/天**

**② 进阶版 · 含复购的月营收**
```
月营收 = F_pass × 30 × r1 × r2 × r3 × AOV × (1 + (r4−1)×0.3)
```
> 把复购率高于 1 的部分按 30% 系数加成（避免重复计数）。

**③ 专业版 · 分客群分漏斗**
```
营收 = Σ_segment (F_seg × r1_seg × r2_seg × AOV_seg)
```
天府 IFC 客群细分：
- 金融白领（45%）：进店 15%，转化 75%，AOV 60
- 周边居民（30%）：进店 8%，转化 55%，AOV 40
- 商旅访客（15%）：进店 12%，转化 65%，AOV 80
- 其他（10%）：进店 5%，转化 40%，AOV 30

## ④ 竞争生态层（Competition Ecosystem）

**含义**：扎堆不一定是坏事——同业聚集吸客（集聚效应），异业互补提单（消费链）。

### 子指标 & 数据来源

| 子指标 | 符号 | 数据来源 | 天府 IFC 取值 |
|---|---|---|---|
| 同品类门店数（500m 内） | N_same | PDF 餐饮 2092、零售 2309 等 | 咖啡同业约 35 家 |
| 同品类门店质量评分 | Q_same | 大众点评 AI 均值估算 | 4.2 ★均值 |
| 互补业态数 | N_comp | PDF 业态分布 | 写字楼+商场+酒店共 500+ |
| 锚店权重 | W_anchor | 是否有 SKP/in99/万达 | 银泰 in99 = 0.95 |
| 业态饱和度 | σ | N_same / 商圈承载量 | 咖啡 σ≈0.7（接近饱和但未崩） |

### 公式变体

**① 基础版 · 净竞争系数**
```
K = 1 − σ × 0.6 + (N_comp / N_same) × 0.05 + W_anchor × 0.2
```
> 取值范围约 0.5-1.4，作为乘数修正前面算出的营收。
> 天府 IFC 咖啡算例：1 − 0.7×0.6 + (500/35)×0.05 + 0.95×0.2 ≈ **0.485 + 0.714 + 0.19 = 1.39**

**② 进阶版 · 集聚-竞争双效应曲线（U 形）**
```
集聚收益 = log(1 + N_same) × 0.15              # 集聚带来吸客
竞争损耗 = (N_same / N_carry)² × 0.8           # 超载惩罚
K = 1 + 集聚收益 − 竞争损耗 + W_anchor × 0.2
```
> N_carry = 商圈最大承载同业数（如 50 家咖啡）
> 提供"开第几家最合适"的最优解答案。

**③ 专业版 · 引入差异化定位**
```
K = (1 − σ_直接) × (1 + σ_互补 × 0.3) × (1 + 差异化分 × 0.2)
```
差异化分 = (你的定位向量 与 现有门店定位向量的余弦距离均值) × 10

## ⑤ 外部性（Externality）

**含义**：不可控的"风向"。政策、舆情、城市规划。这一项主要是**修正系数**，不直接产数。

### 子指标 & 数据来源

| 子指标 | 符号 | 数据来源 | 天府 IFC 取值 |
|---|---|---|---|
| 政策红利评分（-2 ~ +2） | π_pol | 自贸区/金融城规划文件 | **+1.5**（成都金融总部基地） |
| 在建竞品风险（-2 ~ 0） | π_risk | 金融城 8 号（建设中） | -0.5 |
| 媒体曝光指数（0-10） | π_media | 微博/小红书提及量 | 8.2 |
| 社会舆情情感（-1 ~ +1） | π_sent | NLP 情感分析 / AI | +0.6 |
| 季节/节庆波动 | π_season | 月度修正系数 | 春节-30%、双 11+20% 等 |

### 公式变体

**① 基础版 · 简单加成**
```
X = 1 + 0.05·π_pol + 0.03·π_risk + 0.01·π_media + 0.05·π_sent
```
> 取值约 0.85-1.20。天府 IFC 算例：1 + 0.075 − 0.015 + 0.082 + 0.03 ≈ **1.17**

**② 进阶版 · 含季节调整**
```
X_month = X_base × (1 + π_season_m)
```
> 12 个月输出 12 个 X 值，沙盘可加"月份滑块"切换。

**③ 专业版 · 蒙特卡洛风险情景**
对每个 π 给一个分布（如政策 ±0.3 正态），跑 1000 次模拟，输出：
- ROI 中位数、P10、P90
- 破产概率（ROI < 0 的次数 / 1000）

## 综合输出公式（4 种选项）

### Output ①：综合评分（最简）
```
Score = 0.25·G + 0.20·E + 0.20·C + 0.20·K + 0.15·X    // 各项已归一化到 0-10
```

### Output ②：预测年营业额
```
年营收 = 日营收(③漏斗) × K(④竞争修正) × X(⑤外部修正) × 365
```

### Output ③：租金比
```
租金比 = (月租金 × 12) / 年营收
天府 IFC 算例：100.2 × 200 × 12 / 年营收
健康线：餐饮 < 18%，零售 < 12%
```

### Output ④：投资回报率 ROI / 回收期
```
ROI = (年营收 × 净利率 − 年租金 − 年人工 − 年其他) / 初始投资
回收期 = 初始投资 / 年净利
```
> 净利率经验值：咖啡 15%、简餐 8%、轻奢零售 25%
